# Plan_D - 2025-1 操作系统原理编程实践项目
## 实现目标
基于HarmonyOS操作系统，实现一个应用程序。要求调用HarmonyOS提供的API，实现应用的功能。

## 项目介绍
> 该应用只能在 HarmonyOS Next 操作系统上运行。

本项目基于HarmonyOS Next操作系统，实现了一个日期规划类型的应用程序。初步实现了以下的功能：
- 界面设计：采用了HarmonyOS提供的组件库，简单地实现了应用的界面设计。
- 待办事项清单：用户可以添加待办事项，并设置截止日期/描述/是否已完成等属性。
- 短时规划： 用户可以通过待办事项清单中的项目直接规划一段时间的任务，并实现了相应的倒计时界面。
- 使用了存储功能，用户的待办事项数据和正在进行的短时规划可以自动保存到本地，下次打开应用时可以直接恢复。

本项目使用 ArkTS 作为主语言，使用 DevEco Studio 作为开发工具。

ArkTS 是 HarmonyOS 官方提供的TypeScript语言的扩展，可以更方便地调用HarmonyOS提供的API。更多相关的介绍请参考 [华为开发者学堂](https://developer.huawei.com/consumer/cn/training/) 的在线课程板块。

DevEco Studio 是华为开发者开发工具，可以帮助开发者快速开发、调试、部署HarmonyOS应用。

## 项目结构
Plan_D 使用了层级模型，从上往下分别是：
- 用户界面层：主要负责界面的设计和布局，使用了HarmonyOS提供的组件库。
- 页面逻辑层：主要负责应用的业务逻辑，调用下层包装的各种服务API，以实现包括待办事项的添加、删除、修改、查询等在内的功能。
- 服务逻辑层：负责服务的逻辑处理，包括状态机、调用处理等。
- 数据存储层：负责数据的异步存储和读取。
